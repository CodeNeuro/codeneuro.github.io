NEUROFINDER:
source extraction challenge

A community-driven competition to extract neuronal signals from calcium imaging data




Sharing data sets vetted for training on Amazon S3 with contributions from several labs

Integrating a diversity of community-developed algorithms into a common framework

Running and benchmarking easily and quickly using cloud-based cluster computing


About




We are developing a standardized API for “source extraction” algorithms along with a standard input and output format in the Thunder library, running on Apache Spark.

We will coordinate and launch a competition as a community at the CodeNeuroNYC hackathon, to proceed over subsequent months.

Training data sets will be made available and accessible with cloud-based tools.

Have an algorithm to contribute?
Or want to help with the competition?

Input format


images
- a folder of flat binary files

images/time_t.bin
- file with volume at a single time point

images/conf.json
{
    "dims": [x,y],
    "dtype": "uint16"
}

Output format


sources/sources.json
[
    {
        "id" : 1,
        "coordinates": [[x0,y0], [x1,y1], ...],
        "values": [v0, v1],
        "bbox" : [xx,xy,yx,yy],
        "centroid" : [x,y]
    },
    ...
]
